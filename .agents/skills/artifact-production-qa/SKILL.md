# Artifact Production and Quality Assurance

## Purpose

Deliver usable files, not merely pasted text. Preserve the requested format and verify the final artifact visually and structurally.

## Workflow

1. Identify the artifact type, audience, template, page/slide constraints, branding, and required source material.
2. Inspect any provided file before editing. Preserve styles, section breaks, formulas, tables, citations, and layout unless the user requests redesign.
3. Use the platform's dedicated document, PDF, spreadsheet, or slide workflow rather than improvised plain-text conversion.
4. Build content with a clear hierarchy, consistent terminology, and appropriate professional tone.
5. Validate the artifact:
   - **DOCX:** headings, styles, margins, page breaks, tables, headers/footers, references, tracked changes if required
   - **PDF:** page rendering, clipping, font embedding, image quality, links, pagination
   - **Spreadsheet:** formulas, units, cell types, validation, freeze panes, filters, charts, totals, and edge cases
   - **Slides:** narrative flow, legibility, alignment, slide density, image quality, citations, and render/open integrity
6. Re-open or render the final output and correct defects before delivery.
7. Provide the actual downloadable file link and a brief note of what was produced.

## Content safeguards

- Do not fabricate missing data, references, logos, signatures, approvals, or results.
- Do not expose hidden metadata, credentials, patient identifiers, or confidential comments.
- Use tables only when they improve comprehension.
- Keep medical and research terminology technically accurate.
- For official submissions, match the supplied institutional template rather than substituting a generic design.

## Validation evidence

Completion requires evidence appropriate to the format, such as successful file parsing, formula inspection, rendered page/slide images, or opening the generated package without repair warnings.

## Failure and uncertainty handling

When a requested template, font, source image, or required datum is unavailable, use a clearly marked placeholder or state the limitation. Do not claim exact visual fidelity without rendering or inspection.
